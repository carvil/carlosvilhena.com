# CLAUDE.md — carlosvilhena.com

Personal blog of Carlos Vilhena, published as **"Tiny Delights"**. Migrated from WordPress (HostGator Hatchling, McLuhan theme) to a static Astro site on Cloudflare Pages.

## Branding — two-layer

- **Publication: "Tiny Delights"** — surfaces in: homepage hero h1, browser tab title, og:site_name, WebSite JSON-LD `name`, RSS feed title, `/llms.txt` header.
- **Author: "Carlos Vilhena"** — surfaces in: navbar (top-left home link), footer copyright, Article JSON-LD `author`, Person JSON-LD `name`, byline.
- Don't collapse them. They serve different surfaces and SEO/AI both publication and author identities to be discoverable separately.

## Stack

- **Astro 6** — starting point: forked from [brook-2](https://github.com/holger1411/astro-brook) (MIT). Attribution in `README.md`.
- **Tailwind CSS v4** + custom CSS in `src/styles/global.css`.
- **MDX** for posts (so we can use the `<Aff>` component inline).
- **`@astrojs/sitemap`** + **`@astrojs/rss`** wired by brook-2.
- **`astro-recommends`** (sister library at `~/Documents/Personal/astro-recommends/`) handles affiliate cloaking. Linked locally during dev via `pnpm link` (or `link:` workspace).
- Deploy target: **Cloudflare Pages**.
- Analytics: **Cloudflare Web Analytics** (free, no consent banner needed in most cases). Replaces Google Analytics + Jetpack stats.
- Email capture: **Kit (ConvertKit)** form embed in a layout component.

## Conventions

### URLs (preserve everything)
- Posts live at `/<slug>/` — flat. Brook-2 ships posts at `/posts/<slug>/`; we **flatten** during adaptation.
- Pages: `/about-me/`, `/now/`, `/get-in-touch/`.
- Categories: `/leadership-breadcrumbs/`, `/stem-education/`, `/side-quests/`. Old `/category/<slug>/` paths get 301 redirects in `_redirects`.
- Affiliate cloak: `/go/<slug>/` → real destination (handled by `astro-recommends`). **Never** rename to `/recommends/`.

### Categories (3, fixed)
| Slug | Display |
|---|---|
| `leadership-breadcrumbs` | Leadership Breadcrumbs |
| `stem-education` | STEM Education |
| `side-quests` | Side Quests |

`Uncategorized` from WP is dropped during migration. `category` is a single-value enum in frontmatter; `tags` may exist alongside as free-form taxonomy (brook-2's tag pages stay).

### Dates — implicit only
- `publishedAt` lives in frontmatter. Used for **sorting** (most recent first) on homepage and category pages.
- Used in **JSON-LD** `Article.datePublished` and **RSS** `<pubDate>` so search engines and feed readers see real dates.
- **NEVER rendered visibly** in the UI. No "May 2, 2026" headings, no relative times, no year groupings on listings. Goal: timeless feel, ordered freshness without visible staleness.
- `updatedAt` optional, same treatment.

### Images — co-located per post
Each post is a folder: `src/content/posts/<slug>/index.mdx` + assets in the same folder. Markdown image refs use relative paths (`./cover.jpg`) so Astro's image pipeline (sharp) optimizes at build. **Do not** use `public/images/` for post images — that bypasses optimization. Brook-2's default `public/images/` pattern is refactored away.

### Affiliate links
Use `<Aff slug="...">label</Aff>` from `astro-recommends`. The library generates `_redirects` for Cloudflare Pages from `affiliates.config.ts`. Defaults: `rel="sponsored nofollow noopener"`, `target="_blank"`. Cloak prefix is `/go/`.

## Commands

```bash
pnpm install
pnpm dev          # astro dev — http://localhost:4321
pnpm build        # astro build — output in dist/
pnpm preview      # serve dist/
```

## Content scope

15 posts, 4 pages, 71 affiliate links migrated from the WordPress site. Source extraction notes live alongside the migration scripts (when written).

## Deploy

Cloudflare Pages, build command `pnpm build`, output `dist/`. `_redirects` is generated by `astro-recommends` at build time and lands in `dist/`.

## Out of scope (intentionally)

- No comments. Brook-2 doesn't ship them; we don't add them.
- No ads. Removed during migration.
- No Yoast/SEO plugin equivalent — JSON-LD + sitemap + RSS + meta tags are wired by hand in `BaseLayout.astro` and `PostLayout.astro`.
